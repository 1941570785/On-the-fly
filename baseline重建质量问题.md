# On-the-fly NVS Baseline 渲染问题分析

## 分析范围

本文档基于原论文 baseline 在以下结果目录中的测试渲染图进行人工对比分析：

- `results/StaticHikes/*`
- `results/TUM/*`

对比对象为对应数据集原始输入图像：

- `datasets/StaticHikes/*/images/*`
- `datasets/TUM/*/images/*`

目标是总结当前 baseline 在 `StaticHikes` 和 `TUM` 上的主要视觉失败模式，为后续创新方向提供依据。

## 总体结论

虽然当前复现的整体指标相较论文报告值并不差，部分场景甚至还有提升，但视觉质量仍然存在明显问题，尤其体现在以下两个方面：

1. `StaticHikes`：长序列后期明显漂移，远景与中景细节被严重平滑，植被和碎石区域出现明显的模糊、串层和局部漂浮伪影。
2. `TUM`：室内场景中小物体、桌面边缘和遮挡边界容易出现浮影、糊边和局部几何错误，导致局部区域看起来像“重影”或“半透明覆盖”。

这些问题说明当前 baseline 的主要瓶颈并不只是平均 PSNR，而是：

- 长序列位姿和几何稳定性不足
- 动态或半动态区域缺少鲁棒处理
- 低纹理和高频边缘区域的高斯初始化与优化不够稳定
- 测试视角下的局部视图一致性较差

## StaticHikes 的主要问题

### 1. 大范围水彩化和平滑模糊

在 `forest1` 和 `forest2` 的中后期帧中，最明显的问题是整幅图像被过度平滑，细节像被“抹掉”。

典型示例：

- 渲染：`results/StaticHikes/forest1/test_images/001080.jpg`
- GT：`datasets/StaticHikes/forest1/images/001080.jpg`
- 渲染：`results/StaticHikes/forest1/test_images/001190.jpg`
- GT：`datasets/StaticHikes/forest1/images/001190.jpg`
- 渲染：`results/StaticHikes/forest2/test_images/000920.jpg`
- GT：`datasets/StaticHikes/forest2/images/000920.jpg`

主要现象：

- 岩石坡面的大量高频碎石纹理被抹成连续灰块
- 树木轮廓、枝叶层次消失，边缘被大面积低通化
- 路径、灌木、碎石之间的结构分离度明显降低

这类现象通常意味着：

- 长序列位姿累计误差导致多视角无法精确对齐
- 高斯表示用平滑颜色场去“平均解释”错位区域

### 2. 前景与背景深度串层

典型示例：

- 渲染：`results/StaticHikes/forest1/test_images/001190.jpg`
- GT：`datasets/StaticHikes/forest1/images/001190.jpg`

主要现象：

- 前景植被与远处石坡相互污染
- 局部区域像覆盖了一层半透明雾状结构
- 景物前后层次关系不清晰，近景物体没有稳定遮挡背景

这类问题更像是：

- 位姿已经漂移
- 新高斯深度初始化偏差较大
- 动态植被或轻微摆动区域被错误当成静态几何写入场景

### 3. 图像边缘和植被区域存在脏污式伪影

典型示例：

- 渲染：`results/StaticHikes/forest1/test_images/000000.jpg`
- GT：`datasets/StaticHikes/forest1/images/000000.jpg`

主要现象：

- 画面四周有明显黑边、暗边或模糊残影
- 顶部枝叶位置出现不自然的黑色拖影
- 边缘像被错误的前景高斯覆盖

这类问题说明：

- 早期建图阶段对边界区域和细枝叶的建模不稳定
- 前景高斯与背景几何关系没有处理好

### 4. 长序列后期质量明显崩塌

从 `results/StaticHikes/forest1/keyframe_metrics.csv` 可以看出，`forest1` 不是“整体一直差”，而是随着序列推进逐渐恶化。

代表性低谷：

- `001080.jpg` 对应测试帧 PSNR 约 `11.35`
- `001190.jpg` 对应测试帧 PSNR 约 `14.97`

而前中期很多帧仍能保持在 `18-24 dB` 左右。

这说明：

- 场景一开始并非无法重建
- 主要问题出现在长序列累计误差
- 是典型的 on-the-fly 场景下“局部稳定、全局逐步失稳”

### 5. university2 相对更稳，但细节仍明显发软

典型示例：

- 渲染：`results/StaticHikes/university2/test_images/000600.jpg`
- GT：`datasets/StaticHikes/university2/images/000600.jpg`

主要现象：

- 台阶边缘、砖墙纹理、栏杆直线结构被软化
- 右侧植被边界出现拖影和细节丢失
- 全局结构尚可，但局部看起来像“过平滑重建”

这说明 `university2` 的主要问题不是大漂移，而是：

- 边缘高频细节保持不足
- 局部结构一致性不够强

## TUM 的主要问题

### 1. 室内小物体和边缘区域明显浮影

典型示例：

- 渲染：`results/TUM/long_office_household/test_images/1341848032.302842.png`
- GT：`datasets/TUM/long_office_household/images/1341848032.302842.png`

主要现象：

- 桌面物体边界被拉花
- 电钻、显示器、桌边、玩偶等位置出现明显重影
- 物体之间像有一层错误透明度叠加

这种问题更像是：

- 局部几何不稳定
- 小尺度高斯漂浮在正确物体附近
- 视角变化后局部遮挡关系无法稳定解释

### 2. 小物体、细长结构和高对比边界容易塌陷

典型示例：

- 渲染：`results/TUM/long_office_household/test_images/1341848050.534849.png`
- GT：`datasets/TUM/long_office_household/images/1341848050.534849.png`

主要现象：

- 骰子、书本、剪刀、瓶子边缘发虚
- 桌面直线和物体矩形边界出现软化
- 局部区域有轻微“薄雾化”或模糊融合

这说明在 TUM 上的主要问题是：

- 室内弱纹理平面和小物体的几何排序不稳定
- 高斯优化容易把边缘局部平均化

### 3. 中后段关键帧质量波动明显

从 `results/TUM/long_office_household/keyframe_metrics.csv` 可以看出，该场景并非整体都差，而是出现明显波动：

- 一部分帧可以到 `24-32 dB`
- 另一部分帧会掉到 `14-18 dB`

这类现象说明：

- 系统具备较强重建能力
- 但某些局部视角或中后段位姿会突然不稳
- 最终视觉上表现为“均值不错，但坏帧很多”

### 4. desk1 和 desk2 的差异说明问题主要出在稳定性

从结果指标看：

- `desk2` 平均最好，但仍有若干 test frame 会明显掉分
- `desk1` 比 `desk2` 更不稳定，坏帧比例更高

因此 TUM 的核心问题不是“全局不可重建”，而是：

- 位姿初始化和增量跟踪对局部条件敏感
- 小物体和近距离几何边缘很容易暴露问题

## 两类数据集的共同失败模式

### 1. 错位后被颜色场平均化

表现为：

- 细节丢失
- 纹理发糊
- 边界变软

这是当前 baseline 在多视角不完全对齐时最常见的退化形式。

### 2. 动态或高频区域缺少显式鲁棒处理

表现为：

- 植被拖影
- 室内边界浮影
- 小物体轮廓不稳定

### 3. 测试视角下局部一致性不足

表现为：

- 平均指标尚可，但单帧视觉质量差异大
- 某些 test frame 会突然明显崩坏

## 对后续创新的直接启发

基于当前 baseline 的失败模式，后续创新更应优先针对以下方向，而不是只追求平均 PSNR 提升：

1. 长序列漂移抑制
   重点针对 `StaticHikes/forest1` 这种随序列推进逐渐失稳的问题。

2. 动态植被和高频边缘鲁棒建模
   重点减少枝叶、灌木、前景植物造成的错误高斯和拖影污染。

3. 不确定性感知的高斯生成与保留
   让高斯不要在高不确定区域盲目增殖，也避免错误几何长期保留。

4. 室内小物体和边缘结构保护
   重点改善 `TUM` 上小物体、桌面边界、细长物体的糊边与浮影问题。

5. 测试视角一致性增强
   不仅关注均值指标，还要重点关注坏帧比例和关键帧级稳定性。

## 当前最值得重点攻克的场景

按视觉失败程度和研究价值排序，建议重点关注：

1. `StaticHikes/forest1`
   最典型的长序列漂移和后期崩塌场景。

2. `TUM/long_office_household`
   最典型的室内局部浮影、边界糊化和中后段波动场景。

3. `StaticHikes/forest2`
   结构还在，但已出现明显长序列后期模糊和边界发软。

4. `StaticHikes/university2`
   更适合研究“全局结构尚可但局部细节仍差”的情况。

## 简要总结

当前 baseline 在 `StaticHikes` 和 `TUM` 上的视觉问题，本质上可以归结为：

- `StaticHikes`：长序列漂移、动态植被污染、远景碎石与树线细节崩塌
- `TUM`：局部几何错误、小物体糊边、边缘浮影和测试视角不一致

因此，后续如果要做有说服力的创新，最有价值的目标不是“均值指标再高一点”，而是：

- 减少 `StaticHikes` 后期大面积水彩化
- 减少植被拖影和深度串层
- 减少 `TUM` 上的桌面小物体浮影、边界糊边与局部重影
