# On-the-fly NVS 创新规划

## 基线诊断

基线的强项是有序拍摄、局部增量位姿初始化、快速高斯增殖和 anchor 扩展；弱项则几乎都和“局部性太强”有关。

关键证据：

- 关键帧触发基本只依赖相邻帧匹配位移和内点数，见 [/data2/zxd/3D_Reconstruction/On_the_fly/train.py](/data2/zxd/3D_Reconstruction/On_the_fly/train.py)。
- 数据读取对相机模型的支持很窄，`ImageDataset` 只真正按 `PINHOLE/SIMPLE_PINHOLE` 处理内参，见 [/data2/zxd/3D_Reconstruction/On_the_fly/dataloaders/image_dataset.py](/data2/zxd/3D_Reconstruction/On_the_fly/dataloaders/image_dataset.py)。
- 训练损失主要是 `L1 + DSSIM + 单目深度约束`，且深度权重会快速衰减，见 [/data2/zxd/3D_Reconstruction/On_the_fly/scene/scene_model.py](/data2/zxd/3D_Reconstruction/On_the_fly/scene/scene_model.py) 与 [/data2/zxd/3D_Reconstruction/On_the_fly/scene/keyframe.py](/data2/zxd/3D_Reconstruction/On_the_fly/scene/keyframe.py)。
- 新高斯增殖主要由 Laplacian 与当前渲染惩罚驱动，低纹理区域、动态区域、错误深度区域缺少显式不确定性建模，见 [/data2/zxd/3D_Reconstruction/On_the_fly/scene/scene_model.py](/data2/zxd/3D_Reconstruction/On_the_fly/scene/scene_model.py) 与 [/data2/zxd/3D_Reconstruction/On_the_fly/poses/guided_mvs.py](/data2/zxd/3D_Reconstruction/On_the_fly/poses/guided_mvs.py)。
- README 明确承认 `No Loop Closure`、`Pinhole Camera Model`、`Avoid rotation without translation`，见 [/data2/zxd/3D_Reconstruction/On_the_fly/README.md](/data2/zxd/3D_Reconstruction/On_the_fly/README.md)。

现有结果和你的观察高度一致：

- `MipNeRF360/garden` 最终 `PSNR=25.13`，但 `SSIM=0.762`，说明最终像素均值不差，但结构一致性仍不足。
- `TUM/long_office_household` 最终 `PSNR=21.20 / SSIM=0.802`，而关键帧曲线中测试帧波动明显，说明中后段位姿或几何稳定性不足。
- `StaticHikes/forest1` 最差，`PSNR=17.85 / SSIM=0.476 / LPIPS=0.434 / t=0.600`，更像长序列漂移、动态干扰、广角畸变和局部窗口误差叠加。

## 推荐主创新方向

最值得做成论文主线的是：**不确定性感知的在线联合优化**，具体落成一个三部分方法。

1. `Distortion-aware online pose/intrinsics`

针对 `StaticHikes` 这类 `OPENCV` 相机，不再把所有图像强行压成单焦距 pinhole 假设，而是在线维护轻量内参校正或畸变补偿。

1. `Uncertainty-aware Gaussian spawning and pruning`

把新高斯增殖从单纯 `Laplacian - render penalty`，升级为融合多源置信度：匹配内点稳定性、PnP/Mini-BA 重投影残差、单目深度置信度、Guided-MVS 一致性、时序稳定性。高不确定区域延迟增殖或降低学习率，低不确定区域优先增殖与保留。

1. `Drift-aware local-global replay`

保持 on-the-fly 不做完整回环优化，但引入一个轻量的“残差驱动回放池”。当关键帧残差、位姿波动、anchor 边界不一致升高时，优先重放少量历史关键帧和边界视角，做局部一致性修复，而不是均匀随机采样优化。

这条主线的优点：

- 对你现在最差的 `StaticHikes` 和 `TUM` 问题是对症下药。
- 不需要推翻 baseline 框架，仍然可以复用现有 `SceneModel`、`Keyframe`、`PoseInitializer`、`Anchor`。
- 论文 narrative 很完整：从“快但脆弱的局部增量”升级为“快且更稳的置信驱动在线重建”。

## 备选创新方向

如果你想更偏渲染质量而不是位姿鲁棒性，可以考虑以下两条备选线。

### 方向 A：低纹理与远景友好的分层高斯初始化

目标是补 baseline 在平面、远景和薄结构上的欠建模。做法是把高斯初始化拆成“高频边缘快速增殖 + 低频区域延迟确认增殖”，再对远景和薄结构用不同尺度先验。

适合主打数据集：[/data2/zxd/3D_Reconstruction/On_the_fly/datasets/MipNeRF360/garden](/data2/zxd/3D_Reconstruction/On_the_fly/datasets/MipNeRF360/garden)、[/data2/zxd/3D_Reconstruction/On_the_fly/datasets/StaticHikes/forest1](/data2/zxd/3D_Reconstruction/On_the_fly/datasets/StaticHikes/forest1)。

### 方向 B：动态/模糊鲁棒的在线重建

目标是解决你提到的 `StaticHikes` 动态模糊、`TUM` 浮影噪点问题。做法是增加轻量动态区域抑制与时序模糊感知，把动态区域从位姿估计、深度引导和高斯生成中降权。

适合主打数据集：[/data2/zxd/3D_Reconstruction/On_the_fly/datasets/StaticHikes/forest1](/data2/zxd/3D_Reconstruction/On_the_fly/datasets/StaticHikes/forest1)、[/data2/zxd/3D_Reconstruction/On_the_fly/datasets/TUM/long_office_household](/data2/zxd/3D_Reconstruction/On_the_fly/datasets/TUM/long_office_household)。

## 当前最值得优先验证的渲染/重建问题

建议优先围绕以下 4 类问题做可视化和定量诊断。

- `长序列漂移`：重点看 `StaticHikes/forest1` 与 `TUM/long_office_household` 的关键帧测试指标曲线、位姿误差曲线、anchor 切换前后质量波动。
- `相机模型失配`：重点看 `StaticHikes` 的 `OPENCV` 相机模型是否导致边缘区域几何拉扯、重投影残差偏高。
- `动态/模糊污染`：重点看树叶、草丛、遮挡边界、室内重复纹理处是否被错误增殖为高斯，形成噪点、浮影和发糊边界。
- `低纹理欠采样`：重点看墙面、地面、远景区域是否因为采样依赖高频纹理而出现空洞、漂浮点或结构稀疏。

## 建议实验路线

第一阶段先做 baseline 诊断，不改方法。

- 用你新增的 [/data2/zxd/3D_Reconstruction/On_the_fly/scripts/train_with_keyframe_export.py](/data2/zxd/3D_Reconstruction/On_the_fly/scripts/train_with_keyframe_export.py)、[/data2/zxd/3D_Reconstruction/On_the_fly/scripts/keyframe_eval_all.py](/data2/zxd/3D_Reconstruction/On_the_fly/scripts/keyframe_eval_all.py)、[/data2/zxd/3D_Reconstruction/On_the_fly/scripts/plot_keyframe_metrics.py](/data2/zxd/3D_Reconstruction/On_the_fly/scripts/plot_keyframe_metrics.py) 完整导出关键帧曲线。
- 额外记录每关键帧的重投影残差、PnP 内点率、深度对齐误差、anchor 编号和高斯数量变化。

第二阶段做最小可行创新。

- 先实现 `uncertainty-aware spawning/pruning`，因为它最容易接入现有 `SceneModel.add_new_gaussians()`，同时直接作用于质量。
- 再实现 `drift-aware replay scheduling`，替换当前较随机的帧采样优化策略。

第三阶段再做论文强化版。

- 加入 `distortion-aware online intrinsics` 或轻量动态区域抑制，作为第二个核心模块，把方法从“只对有序静态 pinhole 序列有效”推进到“对真实长序列更稳”。

## 预期论文贡献组织

建议最终按 3 个贡献写。

- 一个面向在线重建的不确定性建模模块，统一影响位姿、深度和高斯增殖。
- 一个保持实时预算的残差驱动回放或局部全局混合优化策略。
- 一个针对真实采集退化因素的鲁棒性增强模块，优先考虑畸变相机或动态区域抑制。
